# ESTADO DEL ARTE

**Author (s): Ghousun Ayed Alsharari, Ayman Mohamed Mostafa**

**Títle of paper: Developing an Intrusion Detection System (IDS) For Network Security Using Machine Learning**

**Journal: International Journal of Intelligent Systems and Applications in Engineering**

**Volume (issue): 12(4)**

**pag – pag (year): 2005 – 2026 (2024)**

## Estado del arte que hace el autor

Los autores desarrollan una investigación centrada en la construcción de un Intrusion Detection System orientado a la seguridad de redes IoT utilizando técnicas de Machine Learning y Deep Learning. El artículo parte del crecimiento acelerado del Internet of Things y de cómo la expansión de dispositivos conectados incrementa la superficie de ataque dentro de redes industriales, médicas y domésticas. Los autores explican que la arquitectura IoT está conformada por capas de percepción, red y aplicación, las cuales permiten el intercambio constante de información pero también generan vulnerabilidades frente a amenazas como Data Breaches, Man-in-the-Middle Attacks y Denial of Service. El estudio revisa conceptos relacionados con Host-based IDS, Network-based IDS y sistemas híbridos, además de enfoques de detección basados en firmas, anomalías y especificaciones. Asimismo, los autores realizan una revisión extensa de investigaciones previas relacionadas con RNN, CNN, ANN, SVM, Decision Trees, Random Forest, Bayesian Networks y técnicas híbridas de detección de intrusiones. El artículo también analiza limitaciones presentes en estudios anteriores, como overfitting, problemas de escalabilidad, dificultad para detectar ataques U2R y R2L, dependencia de datasets antiguos y altos costos computacionales. Finalmente, el estado del arte destaca la importancia de feature engineering mediante técnicas como mRMR y PCA para reducir dimensionalidad y mejorar precisión dentro de sistemas IDS modernos.

## Motivación del autor

La motivación principal del artículo surge debido al incremento constante de amenazas cibernéticas dentro de ecosistemas IoT y a las limitaciones de los IDS tradicionales para detectar ataques complejos en tiempo real. Los autores consideran que el crecimiento del IoT industrial, médico y empresarial ha generado enormes volúmenes de tráfico de red difíciles de analizar usando técnicas convencionales. Además, identifican que muchos modelos de Intrusion Detection presentan problemas de precisión, alto consumo computacional y dificultades para adaptarse a nuevos patrones de ataques. Otra motivación importante es demostrar cómo las técnicas modernas de Machine Learning y Deep Learning pueden fortalecer los IDS mediante análisis inteligente de tráfico de red y feature engineering avanzado. Asimismo, el estudio busca comparar distintos modelos ML y DL dentro de un mismo framework para identificar cuáles ofrecen mejor desempeño en ambientes IoT. Los autores también pretenden reducir los costos computacionales asociados a análisis masivos de datos mediante reducción de características y selección inteligente de atributos relevantes.

## Descripción del aporte del autor

El principal aporte del artículo es el desarrollo de un IDS inteligente basado en Machine Learning y Deep Learning diseñado específicamente para ambientes IoT. Los autores proponen un framework que integra técnicas de feature selection mediante mRMR y feature reduction mediante PCA para optimizar el análisis de tráfico de red y mejorar la capacidad de detección de amenazas. Otro aporte importante es la comparación experimental entre múltiples modelos de clasificación, incluyendo SVM, KNN, Naive Bayes, Decision Trees, AdaBoost, Bagging Trees y LSTM. Asimismo, el trabajo aporta un enfoque multinivel capaz de detectar tanto tráfico normal o malicioso como clasificar distintos tipos de ataques específicos. El artículo también presenta un pipeline completo que incluye preprocesamiento, manejo de valores faltantes, one-hot encoding, normalización Min-Max y entrenamiento de modelos. Finalmente, los autores validan experimentalmente el framework utilizando el dataset ToN-IOT, demostrando mejoras significativas en precisión y reducción de complejidad computacional gracias a feature engineering.

## Proceso para obtener el aporte que considera el autor

Para obtener el aporte, los autores desarrollaron una metodología experimental basada en varias etapas. Primero seleccionaron el dataset ToN-IOT, compuesto por más de 461 mil registros y 52 características relacionadas con tráfico IoT industrial. Posteriormente realizaron preprocesamiento de datos, incluyendo manejo de valores faltantes, one-hot encoding para variables categóricas y normalización Min-Max para escalar características numéricas. Luego aplicaron técnicas de feature engineering utilizando PCA para reducción de dimensionalidad y mRMR para selección de características relevantes. Después dividieron el dataset usando Holdout con 70% para entrenamiento y 30% para pruebas. A continuación entrenaron distintos modelos ML y DL como SVM, Decision Trees, Naive Bayes, KNN, AdaBoost, Bagging Trees y LSTM. Finalmente evaluaron el desempeño utilizando métricas de accuracy, precision, recall y F1-score, comparando resultados con y sin técnicas de feature engineering para medir el impacto real de PCA y mRMR.

## Proceso para resolver el problema considerado por el autor

Los autores resolvieron el problema mediante la construcción de un sistema IDS inteligente basado en Machine Learning y Deep Learning optimizado mediante feature engineering. Primero identificaron problemas comunes relacionados con alta dimensionalidad, redundancia de datos y costos computacionales elevados dentro de ambientes IoT. Posteriormente aplicaron PCA para reducir el número de características preservando la mayor cantidad de información posible. Después utilizaron mRMR para seleccionar únicamente atributos relevantes y no redundantes. Además, implementaron procesos de limpieza y normalización de datos para mejorar el rendimiento de los modelos. Luego entrenaron distintos algoritmos de clasificación capaces de identificar tráfico normal y múltiples tipos de ataques. Finalmente compararon el rendimiento de cada modelo utilizando métricas estandarizadas y demostraron que el uso combinado de ML, DL y feature engineering mejora significativamente la precisión del IDS mientras reduce requerimientos computacionales.

## Métricas que el autor usa y resultado que obtiene. Comentar

El estudio utiliza métricas clásicas de clasificación como accuracy, precision, recall y F1-score para evaluar el desempeño de los modelos IDS. También emplea indicadores derivados de la matriz de confusión como True Positive, False Positive, True Negative y False Negative. Los autores realizan pruebas comparativas entre distintos algoritmos ML y DL aplicando diferentes técnicas de feature engineering. Los resultados muestran que la combinación de PCA y mRMR mejora notablemente la capacidad de detección y reduce costos computacionales. Asimismo, los modelos entrenados lograron detectar tráfico malicioso y múltiples tipos de ataques dentro del dataset ToN-IOT con altos niveles de precisión. En mi opinión, las métricas utilizadas son adecuadas porque permiten medir no solo exactitud sino también capacidad de detección de amenazas. Además, el hecho de comparar múltiples modelos fortalece mucho la validez experimental del estudio. Sin embargo, el artículo podría complementarse con métricas relacionadas con tiempo de inferencia o rendimiento en entornos IoT reales.

## Observaciones y/o críticas suyas al artículo

El artículo resulta bastante sólido porque combina Machine Learning, Deep Learning y técnicas modernas de feature engineering dentro de un framework práctico de Intrusion Detection para IoT. Uno de sus puntos más fuertes es la extensa revisión bibliográfica y comparativa de investigaciones previas relacionadas con IDS. También destaca positivamente el uso de datasets modernos como ToN-IOT, evitando la dependencia exclusiva de datasets antiguos como KDD99. Además, el trabajo explica claramente cada etapa metodológica, desde preprocesamiento hasta evaluación de modelos. Sin embargo, el artículo presenta algunas limitaciones. Gran parte de las pruebas se realizan sobre datasets controlados y no en ambientes IoT reales, por lo que la generalización práctica podría variar. Asimismo, algunos modelos Deep Learning podrían implicar altos requerimientos computacionales difíciles de implementar en dispositivos IoT limitados. A pesar de ello, el estudio constituye un antecedente muy importante para investigaciones relacionadas con Intrusion Detection, IoT Security y aplicaciones de inteligencia artificial en ciberseguridad.
