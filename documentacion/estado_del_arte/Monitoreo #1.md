# ESTADO DEL ARTE

**Author (s): Zhenyue Long, Huiru Yan, Guiquan Shen, Xiaolu Zhang, Haoyang He, Long Cheng**

**Títle of paper: A Transformer-based network intrusion detection approach for cloud security**

**Journal: Journal of Cloud Computing: Advances, Systems and Applications**

**Volume (issue): Vol. 13 No. 5**

**pag – pag (year): 1 – 11 (2024)**

## Estado del arte que hace el autor

Los autores desarrollan una investigación enfocada en la detección de intrusiones de red dentro de entornos cloud utilizando modelos basados en Transformer. El artículo analiza cómo el crecimiento acelerado del cloud computing ha incrementado la necesidad de mecanismos avanzados de protección frente a amenazas dinámicas y ataques complejos sobre recursos accesibles mediante Internet. Los autores explican que los sistemas tradicionales de seguridad como firewalls no son suficientes para detectar ataques sofisticados como DoS, DDoS, spoofing, ataques man-in-the-middle y otras amenazas avanzadas presentes en entornos cloud. El estudio revisa conceptos relacionados con Network Intrusion Detection Systems, aprendizaje automático, deep learning y arquitecturas Transformer aplicadas a ciberseguridad. Asimismo, se analiza cómo modelos tradicionales basados en RNN y CNN presentan limitaciones relacionadas con secuencias largas, entrenamiento serial y pérdida de características temporales. El trabajo también revisa investigaciones previas relacionadas con intrusion detection, anomaly detection y modelos basados en Graph Neural Networks, LSTM y técnicas de machine learning. Finalmente, los autores justifican el uso del mecanismo de atención de Transformer como alternativa más eficiente para aprender correlaciones temporales y características complejas dentro de tráfico de red.

## Motivación del autor

La motivación principal del artículo surge debido al incremento constante de amenazas cibernéticas en entornos cloud y a las limitaciones de los sistemas tradicionales de detección de intrusiones. Los autores consideran que los firewalls tradicionales no pueden detectar adecuadamente ataques sofisticados y que muchos modelos de deep learning requieren grandes volúmenes de datos de alta calidad para funcionar correctamente. Otra motivación importante es que modelos basados en RNN presentan problemas relacionados con entrenamiento secuencial lento y dificultades para capturar dependencias temporales largas dentro de secuencias de tráfico de red. Además, el artículo busca demostrar que el mecanismo de atención de Transformer puede mejorar significativamente la capacidad de detección de intrusiones dentro de entornos cloud al capturar relaciones complejas entre características de tráfico y tipos de ataques. También existe una motivación orientada a construir sistemas de detección más precisos, eficientes y adaptables frente a amenazas modernas relacionadas con servicios cloud y grandes volúmenes de tráfico de red.

## Descripción del aporte del autor

El principal aporte del artículo es el desarrollo de un modelo de Network Intrusion Detection System basado en arquitectura Transformer específicamente diseñado para entornos cloud. Los autores proponen un algoritmo que integra mecanismos de atención para analizar relaciones entre características de tráfico de red y distintos tipos de ataques, mejorando la precisión del proceso de detección. Otro aporte importante es la adaptación de la arquitectura Transformer para resolver problemas de clasificación de intrusiones, eliminando componentes innecesarios del decoder y utilizando únicamente la estructura encoder junto con capas fully connected y Softmax. Además, el trabajo presenta una arquitectura completa de intrusion detection dividida en etapas de preprocesamiento, entrenamiento y evaluación del modelo. También aporta técnicas de label encoding, normalización de datos y batch processing utilizando PyTorch. Finalmente, los autores validan experimentalmente el modelo utilizando el dataset CIC-IDS 2018, demostrando niveles de accuracy superiores al 93%, comparables con modelos CNN-LSTM.

## Proceso para obtener el aporte que considera el autor

Para obtener el aporte, los autores desarrollaron una metodología basada en deep learning utilizando arquitectura Transformer aplicada a intrusion detection. Primero realizaron un análisis del problema de seguridad en entornos cloud y revisaron investigaciones relacionadas con NIDS, RNN, CNN y modelos de atención. Posteriormente diseñaron un modelo basado únicamente en el encoder de Transformer para adaptarlo a tareas de clasificación de intrusiones de red. Después implementaron mecanismos de multi-head attention, feedforward neural networks y layer normalization. Luego realizaron el preprocesamiento del dataset CIC-IDS 2018 aplicando label encoding, normalización mediante MinMaxScaler y batch processing utilizando DataLoader de PyTorch. Posteriormente entrenaron el modelo utilizando Adam optimizer y CrossEntropyLoss como función de pérdida. También realizaron múltiples pruebas variando el número de capas encoder para evaluar rendimiento y convergencia. Finalmente compararon resultados utilizando métricas como accuracy, precision, recall y F1-score.

## Proceso para resolver el problema considerado por el autor

Los autores resolvieron el problema diseñando un sistema de detección de intrusiones basado en Transformer que aprovecha mecanismos de atención para capturar relaciones temporales complejas dentro del tráfico de red. Primero realizaron el preprocesamiento del dataset CIC-IDS 2018 convirtiendo etiquetas categóricas en vectores numéricos y normalizando datos para mejorar el entrenamiento del modelo. Posteriormente utilizaron una arquitectura Transformer basada únicamente en encoder, eliminando el decoder debido a que intrusion detection es un problema de clasificación y no de generación secuencial. Después implementaron mecanismos multi-head attention que permiten asignar pesos dinámicos a características relevantes del tráfico de red. Además, integraron capas feedforward y layer normalization para mejorar convergencia y estabilidad del entrenamiento. Luego entrenaron el modelo mediante forward propagation y backpropagation utilizando Adam optimizer y CrossEntropyLoss. Finalmente evaluaron el sistema mediante métricas de desempeño y comparaciones experimentales frente a otros modelos de detección.

## Métricas que el autor usa y resultado que obtiene. Comentar

El estudio utiliza métricas clásicas de evaluación de modelos de clasificación, incluyendo accuracy, precision, recall y F1-score. Los autores también utilizan indicadores relacionados con TP, TN, FP y FN para medir correctamente el rendimiento del sistema de detección de intrusiones. Además, realizan pruebas variando parámetros como número de capas encoder, tamaño de batch y dimensiones de atención dentro de la arquitectura Transformer. Los resultados experimentales muestran que el modelo propuesto alcanza niveles de accuracy superiores al 93%, obteniendo un rendimiento comparable con modelos CNN-LSTM pero con mejores capacidades para capturar dependencias temporales mediante mecanismos de atención. Asimismo, los autores demuestran que el modelo puede detectar múltiples tipos de ataques presentes en el dataset CIC-IDS 2018, incluyendo DoS, DDoS, botnets y web attacks. En mi opinión, el trabajo tiene una base experimental bastante sólida porque utiliza datasets reconocidos y métricas ampliamente aceptadas dentro de intrusion detection. Sin embargo, podría fortalecerse realizando pruebas en entornos cloud reales y comparaciones más amplias frente a arquitecturas recientes basadas en Transformers híbridos.

## Observaciones y/o críticas suyas al artículo

El artículo es bastante relevante porque combina tecnologías modernas de deep learning con problemas actuales relacionados con ciberseguridad y cloud computing. Uno de sus puntos más fuertes es la adaptación eficiente de la arquitectura Transformer para intrusion detection, aprovechando mecanismos de atención para mejorar la capacidad de aprendizaje de secuencias temporales dentro del tráfico de red. También destaca positivamente el uso de datasets reconocidos y métricas completas para validar experimentalmente el modelo propuesto. Además, el trabajo explica de forma clara componentes internos de Transformer como multi-head attention, feedforward networks y normalization layers, facilitando la comprensión del modelo. Sin embargo, el artículo presenta algunas limitaciones. La evaluación se realiza principalmente sobre datasets de laboratorio y no sobre tráfico cloud real, lo que podría afectar la generalización del modelo en escenarios productivos. Asimismo, el trabajo no profundiza demasiado en costos computacionales ni consumo de recursos asociados al entrenamiento de modelos Transformer. A pesar de ello, constituye un antecedente muy importante para investigaciones relacionadas con intrusion detection, cloud security y aplicaciones de inteligencia artificial en ciberseguridad.
